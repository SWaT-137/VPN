## 1. Общие положения

### 1.1 Наименование проекта
Разработка программного комплекса для организации защищенного VPN-соединения на основе протокола Trojan с графическим интерфейсом пользователя.

### 1.2 Цель создания
Создание надежного, устойчивого к блокировкам VPN-решения, позволяющего пользователям безопасно подключаться к интернет-ресурсам через сервер, расположенный в Беларуси, с обеспечением высокой скорости соединения и простоты использования.

### 1.3 Задачи проекта
1. Разработка серверной части, реализующей протокол Trojan
2. Разработка клиентской части с графическим интерфейсом на PyQt5
3. Обеспечение шифрования трафика и маскировки под HTTPS
4. Реализация системы аутентификации пользователей
5. Создание механизмов обхода глубокой инспекции пакетов (DPI)
6. Обеспечение платформенности на Windows

## 4. Детальные требования к реализации протокола Trojan

### 4.1 Формат пакета Trojan

Клиент отправляет серверу данные в следующем формате:

```
[ХЕШ ПАРОЛЯ (56 байт)] + [\r\n] + [ЗАПРОС] + [\r\n\r\n]
```

Где:
- **ХЕШ ПАРОЛЯ:** SHA224 от пароля в hex-формате
- **ЗАПРОС:** Стандартный HTTP-запрос или данные туннеля

### 4.2 Алгоритм работы протокола

#### 4.2.1 Установка соединения (Handshake)
1. Клиент устанавливает TCP-соединение с сервером (порт 443)
2. Происходит TLS-рукопожатие
3. Клиент отправляет хеш пароля
4. Сервер проверяет хеш:
   - Если хеш верный → переход в режим прокси
   - Если хеш неверный → перенаправление на локальный веб-сервер

#### 4.2.2 Передача данных
1. Клиент отправляет запрос к целевому ресурсу
2. Сервер устанавливает соединение с целевым ресурсом
3. Данные передаются в обе стороны до закрытия соединения

### 4.3 Требования к безопасности

| ID | Требование | Описание |
|----|------------|----------|
| SEC-01 | TLS 1.3 | Использование современной версии TLS |
| SEC-02 | Perfect Forward Secrecy | Обеспечение PFS через ECDHE |
| SEC-03 | Защита от повторов | Уникальные nonce для каждого соединения |
| SEC-04 | Безопасное хранение паролей | Хеширование с солью на сервере |
| SEC-05 | Защита от брутфорса | Ограничение числа попыток аутентификации |

## 6. Требования к производительности

### 6.1 Серверная часть
- **Максимальное количество одновременных подключений:** ≥ 100
- **Пропускная способность:** ≥ 100 Мбит/с
- **Время отклика:** ≤ 50 мс (при добавлении к пингу)
- **CPU нагрузка:** ≤ 30% на одно ядро при 50 подключениях

### 6.2 Клиентская часть
- **Потребление RAM:** ≤ 100 МБ
- **Потребление CPU:** ≤ 5% в простое, ≤ 15% при передаче
- **Время запуска:** ≤ 3 секунды
- **Время подключения:** ≤ 2 секунды

## 7. Требования к надежности

| ID | Требование | Описание |
|----|------------|----------|
| REL-01 | Восстановление соединения | Автоматическое переподключение при обрыве |
| REL-02 | Graceful shutdown | Корректное закрытие всех соединений |
| REL-03 | Обработка ошибок | Логирование всех исключений |
| REL-04 | Heartbeat | Проверка активности соединения каждые 30 сек |

## 10. Критерии приемки

Проект считается завершенным при выполнении следующих условий:

1. **Функциональность:**
   - Клиент подключается к серверу с правильным паролем
   - Трафик успешно проходит через VPN
   - Kill Switch работает при обрыве соединения
   - Настройки сохраняются между запусками

2. **Производительность:**
   - Скорость соединения не ниже 80% от прямой
   - Пинг увеличивается не более чем на 50 мс

3. **Стабильность:**
   - Нет падений в течение 72 часов непрерывной работы
   - Корректная обработка всех ошибок

4. **Безопасность:**
   - Шифрование TLS 1.3 работает корректно
   - Неверный пароль ведет на веб-страницу
   - Нет утечек DNS/IP

## 11. Приложения

### Приложение А: Список используемых портов
- 443/TCP - основной трафик VPN
- 80/TCP - веб-сервер для маскировки
- 1080/TCP - SOCKS5 прокси (опционально)

### Приложение Б: Формат конфигурационного файла (client_config.yaml)

```yaml
server:
  address: "belarus.vpn.com"
  port: 443
  password: "user_password"

options:
  auto_connect: true
  kill_switch: true
  start_minimized: false
  mode: "auto"  # auto, all, split

proxy:
  type: "socks5"
  port: 1080
```
