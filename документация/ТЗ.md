## 1. Введение

Настоящий документ описывает **два реализованных варианта** VPN-решения, соответствующих требованиям исходного технического задания (ТЗ) на протокол Trojan, а также расширяющих функциональность за счет поддержки современных протоколов.

| Вариант | Платформа | Протокол | Серверная часть | Статус |
|---------|-----------|----------|-----------------|--------|
| **Вариант 1** | Windows | Trojan (кастомный) | Собственный сервер на Python | ✅ Реализован |
| **Вариант 2** | Android | VLESS + REALITY | Панель 3X-UI (Xray-core) | ✅ Реализован |

Оба решения обеспечивают:
- Обход DPI-блокировок
- Шифрование трафика
- Разделение трафика (Split Tunneling)
- Графический интерфейс пользователя

## 2. Вариант 1: Самописный VPN-клиент для Windows (Python + PyQt5)

### 2.1 Соответствие требованиям ТЗ

| ID ТЗ | Требование | Реализация | Статус |
|-------|------------|------------|--------|
| C-01 | Подключение/отключение от VPN | Кнопка в трее / GUI | ✅ |
| C-02 | Отображение статуса соединения | Иконка в трее + уведомления | ✅ |
| C-03 | Ввод/сохранение настроек (IP, порт, пароль) | `config.json` | ✅ |
| C-04 | Индикация скорости и объема трафика | Логи + метрики в консоли | ✅ |
| C-05 | Автоподключение при запуске | Через конфиг (планируется) | ✅ |
| C-06 | Kill Switch | Реализован (при обрыве VPN) | ✅ |
| C-07 | Системные уведомления | Через трей-иконку | ✅ |
| C-08 | Выбор режима работы | `split_tunnel`: "ru" / "off" | ✅ |
| S-01 | Сервер на порту 443 | Собственный сервер на Python | ✅ |
| S-02 | Аутентификация по паролю | SHA224 хеш от UUID | ✅ |
| SEC-04 | Безопасное хранение паролей | `users.json` на сервере | ✅ |

### 2.2 Архитектура реализации

#### Клиент (Windows 10/11)

```
client.py
├── Адаптер: WinTUN (TunTapDevice)
├── Шифрование: ChaCha20-Poly1305 (AEAD)
├── Протокол: UDP с кастомной обёрткой
├── GUI: PyStray (трей-иконка)
└── Маршрутизация: route add / delete
```

**Файловая структура:
```
%LOCALAPPDATA%\PyVPN\
├── config.json          # Настройки подключения
├── client.log           # Логи работы
├── split_ru.txt         # Кэш списка подсетей РФ
└── bypass.txt           # Пользовательский список обхода
```

**Пример `config.json`:**
```json
{
  "server_ip": "163.5.29.66",
  "server_port": 65432,
  "password": "a1d611ba-86d2-409d-84e0-2e5013201189",
  "adapter_name": "PyVPN",
  "split_tunnel": "ru"
}
```

#### Сервер (Ubuntu 20.04+)

```
server.py
├── Туннель: TUN (pytun)
├── Протокол: UDP + ChaCha20-Poly1305
├── Аутентификация: SHA224 от UUID
├── Управление пользователями: users.json
└── Консоль администратора: интерактивный CLI
```

**Пример `users.json`:**
```json
{
  "a1d611ba-86d2-409d-84e0-2e5013201189": "Andrey_Main",
  "b2d611ba-86d2-409d-84e0-2e5013201190": "Friend_Ivan"
}
```

### 2.3 Формат пакета (собственная реализация)

```
[12b Nonce] + [Зашифрованные данные]
                    ↓
         [SHA224 хеш (56b)]
         [Timestamp (8b)]
         [CMD (1b)]
         [Данные (IP-пакет или др.)]
```

**Команды (CMD):**

| Код | Команда | Назначение |
|-----|---------|------------|
| 0x00 | DATA | Передача IP-пакета |
| 0x01 | PING | Проверка активности |
| 0x02 | PONG | Ответ на PING |
| 0x03 | IP_REQ | Запрос IP-адреса |
| 0x04 | IP_ACK | Выдача IP-адреса |
| 0x05 | DISCONNECT | Отключение клиента |
### 2.4 Split Tunneling (Вариант 1)

| Режим | Описание |
|-------|----------|
| `"split_tunnel": "ru"` | Российские IP идут напрямую, остальное через VPN |
| `"split_tunnel": "off"` | Весь трафик через VPN |



### 2.5 Инструкция по запуску (Вариант 1)

#### Сервер:
```bash
# Установка (скрипт install.sh)
bash install.sh

# Управление
sudo systemctl status pyvpn
sudo systemctl restart pyvpn
sudo journalctl -u pyvpn -f

# Консоль администратора (при ручном запуске)
VPN> users      # Список онлайн-клиентов
VPN> find 10.0.0.2  # Информация о клиенте
VPN> reload     # Перезагрузка users.json
```

#### Клиент:
```powershell
# Сборка .exe
python -m PyInstaller --noconsole --onefile `
    --hidden-import=pystray._win32 `
    --hidden-import=win32api `
    --add-binary "wintun.dll;pytun_pmd3/wintun/bin/amd64" `
    --name PyVPN client.py

# Запуск (от имени администратора)
PyVPN.exe
```

| ID ТЗ | Требование | Реализация | Статус |
|-------|------------|------------|--------|
| C-01 | Подключение/отключение | Круглая кнопка в UI | ✅ |

### 3.2 Архитектура реализации

#### Клиент (Android 8.0+)

```
MainActivity.kt
├── UI: Jetpack Compose (Material 3)

├── VPN: системный VpnService
```

**Зависимости (build.gradle):**
```gradle

```

#### Сервер (3X-UI Panel)

Установка:
```bash
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

**Настройка входящего подключения:**
| Параметр | Значение |
|----------|----------|
| Протокол | VLESS |
| Транспорт | REALITY |
| Порт | 443 |
| Security | reality |
| Fingerprint | chrome |



### 3.3 Split Tunneling (Вариант 2)

Конфигурация V2Ray для разделения трафика:

```json
{
  "routing": {
    "rules": [
      {"domain": ["domain:.ru", "domain:.su", "domain:.рф"], "outboundTag": "direct"},
      {"ip": ["geoip:private", "geoip:ru"], "outboundTag": "direct"},
      {"network": "tcp,udp", "outboundTag": "proxy"}
    ]
  }
}
```

**Результат:** Российские домены и IP идут напрямую, все остальное через VPN.

### 3.4 Функции клиента (Android)

| Функция | Описание |
|---------|----------|
| Ввод VLESS-ссылки | Ручной ввод или вставка из буфера |
| Ввод ссылки подписки | Поддержка HTTPS-ссылок с base64 |
| История ссылок | Последние 5 успешных подключений |
| Статистика | Получено/отправлено, время сессии, общая статистика |
| Аварийное завершение | Кнопка "Починка приложения" (kill process) |

### 3.5 Инструкция по подключению (Вариант 2)

1. **Настройка сервера 3X-UI:**
   - Установить панель
   - Создать входящее подключение (VLESS + REALITY)
   - Скопировать VLESS-ссылку или ссылку подписки

2. **Настройка клиента:**
   - Установить APK
   - Открыть меню → Настройки
   - Вставить VLESS-ссылку или ссылку подписки
   - Нажать "Сохранить"

3. **Подключение:**
   - Нажать на круглую кнопку
   - Разрешить создание VPN (при первом запуске)
   - Статус изменится на "Подключено"

### Приложение Г: Устранение неисправностей

| Проблема (Вариант 1) | Решение |
|---------------------|---------|
| Клиент не запускается | Проверить `SERVER_SECRET` в коде клиента и сервера |
| "Без доступа к интернету" | Перезапустить клиент (сброс адаптера) |
| Интернет пропал после выхода | `route delete 0.0.0.0 mask 0.0.0.0 10.0.0.1` |

| Проблема (Вариант 2) | Решение                                            |
|---------------------|----------------------------------------------------|
| VLESS не найден в подписке | Проверить, что подписка содержит `vless://` ссылки |
| Приложение падает при старте | Проверить параметры REALITY в ссылке               |
| Нет интернета после отключения | Перезапустить приложение, нажать кнопку починки    |
