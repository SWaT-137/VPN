

# Документация PyVPN — Стелс-VPN нового поколения

**Версия:** 5.0  
**Назначение:** Обход DPI-блокировок, защита приватности и разделение трафика (Split Tunneling).

PyVPN — это высокотехнологичный VPN-клиент и сервер, написанные на Python. В отличие от классических VPN, PyVPN использует собственный UDP-протокол с шифрованием ChaCha20-Poly1305 и VLESS-подобную аутентификацию, что делает его невидимым для систем Deep Packet Inspection (DPI).

---

## 📖 Архитектура и Принцип работы

### 1. Криптография и Защита
*   **Шифрование канала:** ChaCha20-Poly1305 (AEAD). Обеспечивает конфиденциальность, целостность и аутентичность данных. В отличие от WireGuard или OpenVPN, не имеет стандартных сигнатур, которые DPI умеет распознавать.
*   **Аутентификация:** SHA-224 хеш от UUID пользователя (подобно VLESS/VMess). Сервер мгновенно отбрасывает пакеты с неверным хешем.
*   **Защита от Replay-атак:** Каждый пакет содержит метку времени (Timestamp). Сервер отбрасывает пакеты, время которых отличается от серверного более чем на 30 секунд. Это предотвращает атаки повтором, которые часто применяются цензорами.

### 2. Структура пакета
Вся полезная нагрузка зашифрована. Внутри зашифрованной части лежит:
`[SHA224 Хеш 56b][Timestamp 8b][CMD 1b][IP-пакет]`
Это делает невозможным анализ трафика и выделение сессий без знания ключа.

### 3. Разделение трафика (Split Tunneling)
Встроенная система позволяет пустить трафик российских ресурсов напрямую через твоего провайдера (чтобы банки, Госуслуги и Яндекс работали без проблем), а заблокированные сайты пойдут через VPN.

## 🖥️ Серверная часть (Ubuntu 20.04+)

### 1. Автоматическая установка (Рекомендуется)
Скинь файл install.sh на сервер
```bash
bash install.sh
```
Скрипт автоматически: обновит пакеты, создаст папку `/opt/VnP`, настроит виртуальное окружение Python, установит зависимости, включит форвардинг пакетов, настроит NAT (Masquerade), исправит MTU (TCPMSS), увеличит буферы UDP и создаст службу `systemd` для автозапуска.

### 2. Управление пользователями (`users.json`)
Файл находится по пути `/opt/VnP/users.json`. Формат — словарь, где ключ — это UUID (пароль), а значение — имя пользователя для логов:
```json
{
  "a1d611ba-86d2-409d-84e0-2e5013201189": "Andrey_Main",
  "b2d611ba-86d2-409d-84e0-2e5013201190": "Friend_Ivan"
}
```
*Важно: после изменения файла на сервере, нужно перезагрузить службу или ввести команду `reload` в консоли администратора.*

### 3. Управление службой
```bash
# Статус
sudo systemctl status pyvpn

# Перезапуск (после изменения кода/конфига)
sudo systemctl restart pyvpn

# Логи в реальном времени
sudo journalctl -u pyvpn -f
```

### 4. Консоль администратора
Если запустить сервер вручную (не через systemd), доступна интерактивная консоль:
*   `users` — Показать список онлайн-клиентов и их реальные IP.
*   `find <VPN_IP>` — Подробная информация о клиенте (например, `find 10.0.0.2`).
*   `reload` — Перезагрузить `users.json` без рестарта сервера.
*   `exit` — Остановить сервер.

## 💻 Клиентская часть (Windows 10/11)

### 1. Сборка .exe
Требуется установленный Python, библиотеки и файл `wintun.dll` в папке проекта.

```powershell
python -m PyInstaller --noconsole --onefile --hidden-import=pystray._win32 --hidden-import=win32api --hidden-import=win32gui --add-binary "C:\ПУТЬ_ДО\wintun.dll;pytun_pmd3/wintun/bin/amd64" --name PyVPN client.py
```

### 2. Файловая структура (Портативный режим)
Исполняемый файл `PyVPN.exe` можно запускать из любой папки (Загрузки, Рабочий стол). При первом запуске программа автоматически создаст папку со всеми данными по пути:
`%LOCALAPPDATA%\PyVPN\`

Внутри папки находятся:
*   `config.json` — Настройки подключения.
*   `client.log` — Логи работы (скорость, ошибки, подключение).
*   `split_ru.txt` — Кэш списка подсетей РФ (для Split Tunneling).
*   `bypass.txt` — Пользовательский список обхода (если нужен).

### 3. Настройка `config.json`
Пример конфига:
```json
{
  "server_ip": "163.5.29.66",
  "server_port": 65432,
  "password": "a1d611ba-86d2-409d-84e0-2e5013201189",
  "adapter_name": "PyVPN",
  "split_tunnel": "ru"
}
```
*   `server_ip` / `server_port` — Адрес твоего сервера.
*   `password` — UUID из `users.json` на сервере.
*   `split_tunnel` — Режим разделения трафика (`"ru"` - РФ в обход, `"off"` - весь трафик через VPN).

### 4. Управление через Трей
*   **ПКМ по иконке:** Открыть лог, Открыть папку с конфигом, Выйти.
*   Программа уведомляет о подключении или ошибках через всплывающие уведомления Windows.

## 🌍 Split Tunneling: Разделение трафика

PyVPN умеет пускать российский трафик напрямую через твоего провайдера, чтобы не ломать сайты и экономить скорость VPN-канала.

*   **Как это работает:** При включении (`"split_tunnel": "ru"`) программа скачивает список подсетей РФ с проекта Antifilter (`subnet.lst`, 76 подсетей). Для каждого IP из списка создается маршрут через твоего реального провайдера (с высокой метрикой). Весь остальной трафик уходит в VPN.
*   **Кэширование:** Если при запуске нет интернета или сайт Antifilter недоступен, программа использует вчерашний кэш (`split_ru.txt`).
*   **Кастомный обход:** Создай файл `bypass.txt` в папке `%LOCALAPPDATA%\PyVPN\` и впиши туда свои подсети:
```text
# Моя рабочая сеть
192.168.55.0/24
# IP принтера
10.10.10.5/32
```

## ⚠️ Безопасность и Ограничения

1.  **Ключ шифрования (`SERVER_SECRET`):** В текущей архитектуре ключ шифрования зашит в код (`SWaT_2008` или твой кастомный). Он **ДОЛЖЕН СОВПАДАТЬ** на клиенте (в функции `load_config`) и на сервере. Если ключи разные, сервер не сможет расшифровать пакеты, и VPN не подключится.
2.  **UUID как пароль:** Никогда не используй простые пароли в `users.json`. Генерируй длинные UUID (как в примерах).
3.  **Скорость:** Написан на чистом Python (asyncio). Теоретический потолок скорости — около 30-50 Мбит/с. Для просмотра 4K видео и серфинга этого с запасом хватает, но гигабитных скоростей не будет из-за ограничений языка (GIL).

## 🛠️ Устранение неисправностей

| Проблема | Решение |
| :--- | :--- |
| Клиент не запускается (тихо падает) | Убедись, что `SERVER_SECRET` в `load_config()` совпадает с сервером. Проверь файл `error.log` рядом с `.exe`. |
| "Без доступа к интернету" на адаптере | Нажми правой кнопкой по иконке -> Выход. Запусти заново. Программа сбросит адаптер в DHCP перед настройкой, что очищает кэш Windows. |
| Интернет пропал после выхода из VPN | Редкий баг: маршруты не удалились. Открой `cmd` от админа и выполни: `route delete 0.0.0.0 mask 0.0.0.0 10.0.0.1` |
| Не скачивается список Split Tunneling | Сайт Antifilter лежит. Программа использует кэш (`split_ru.txt`). Если кэша нет, весь трафик пойдет через VPN. |
| Процесс висит в Диспетчере задач | Программа использует `os._exit(0)` для принудительного завершения потоков. Если завис намертво, заверши вручную. |
| Пинги нормальные, но скорость низкая | Проверь буферы UDP на сервере (команда `sysctl net.core.rmem_max` должна выдавать 8388608). Убедись, что на сервере применено правило `TCPMSS --clamp-mss-to-pmtu`. |
