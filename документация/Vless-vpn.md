# Техническая документация VPN-клиентов (Windows & Android)

## 1. Введение

Данный документ описывает два VPN-клиента, использующих протокол **VLESS + REALITY** для обхода блокировок и обеспечения безопасного соединения. Оба клиента используют ядро **Xray** (на Windows) или **V2Ray** (на Android) в качестве транспорта. Подключение осуществляется по ссылкам-подпискам (содержащим base64-кодированный список ссылок) или по прямой VLESS-ссылке.

Клиенты предоставляют графический интерфейс, управление подключением, отображение скорости, трафика и времени сессии.

---

## 2. Общий принцип работы

1. Пользователь вводит **ссылку подписки** (https://…) или **прямую VLESS-ссылку** (vless://…).
2. Приложение извлекает конфигурацию сервера:
   - для подписки: загружает данные, декодирует base64, находит первую VLESS-ссылку.
   - для прямой ссылки: парсит параметры (UUID, адрес, порт, public key, sni, fingerprint и т.д.).
3. Запускается **ядро Xray/V2Ray** с динамически сгенерированной конфигурацией.
4. Настраивается **маршрутизация трафика**:
   - **Windows**: создаётся TUN-адаптер (Xray), добавляются маршруты для всего трафика через TUN, кроме IP-адреса сервера (он идёт через основной шлюз).
   - **Android**: используется системный VPNService, который перехватывает весь трафик и направляет его через локальный SOCKS-прокси (10808), запущенный ядром.
5. После отключения маршруты удаляются / VPNService останавливается.

## 3. Windows-клиент (Python + PySide6)

### 3.1. Компоненты

| Файл | Назначение |
|------|------------|
| `connector.py` | Главный скрипт приложения |
| `xray.exe` | Ядро Xray (должен лежать в папке с программой или в `_MEIPASS` при сборке) |
| `wintun.dll` | Библиотека для работы TUN-адаптера в Windows (рядом с `xray.exe`) |
| `settings.json` | Хранит ссылку подписки |

### 3.2. Требования

- Windows 10 / 11 (x64)
- Установленный Python 3.8+ (только для разработки)
- Права **администратора** (для управления маршрутами и создания TUN-адаптера)

### 3.3. Запуск из исходников

```bash
pip install PySide6
python connector.py
```

При первом запуске приложение запросит ссылку подписки через меню (⋮ → Настройки).

### 3.4. Сборка исполняемого файла (.exe)

Используется **PyInstaller**. Команда:

```bash
pyinstaller --onefile --windowed --add-data "xray.exe;." --add-data "wintun.dll;." connector.py
```

Флаги:
- `--onefile` – один .exe файл.
- `--windowed` – без консоли.
- `--add-data` – включить `xray.exe` и `wintun.dll` в сборку (будут извлечены во временную папку `_MEIPASS`).

### 3.5. Архитектура модулей

#### `resource_path()`
Возвращает абсолютный путь к файлу, корректно работая как в режиме разработки, так и внутри собранного exe (с `sys._MEIPASS`).

#### `run_shell()`, `ps_cmd()`
Утилиты для выполнения команд cmd и PowerShell без окон.

#### `fetch_subscription(url)`
Загружает подписку, декодирует base64 и возвращает список VLESS-ссылок.

#### `parse_vless_link(link)`
Извлекает из VLESS-ссылки все параметры: адрес, порт, UUID, publicKey, shortId, sni, fingerprint, spiderX и т.д.

#### `XrayManager`
Управляет процессом Xray:
- `generate_config()` – создаёт `xray_config.json` с настройками TUN-входящего, исходящего VLESS+REALITY, DNS и маршрутизации.
- `start()` – запускает `xray.exe run -c config.json` скрыто.
- `stop()` – останавливает процесс.

#### `RoutingManager`
Работа с сетевыми маршрутами Windows:
- `get_tun_index()` – получает индекс TUN-адаптера (имя `XrayVPN`).
- `get_main_adapter_info()` – находит основной сетевой интерфейс и его шлюз.
- `add_routes(server_ip)` – добавляет маршрут до IP сервера через основной шлюз, а два маршрута `0.0.0.0 mask 128.0.0.0` и `128.0.0.0 mask 128.0.0.0` через TUN-адаптер. Отключает IPv6 на TUN.
- `remove_routes(server_ip)` – удаляет добавленные маршруты.

#### `ConnectionThread` (QThread)
Асинхронно выполняет подключение: парсит ссылку, генерирует конфиг, запускает Xray, настраивает маршруты. Сигналы `success` / `error` / `status_msg`.

#### `StatsThread` (QThread)
Раз в секунду:
- Измеряет ping до 1.1.1.1 или 8.8.8.8.
- Через PowerShell получает статистику байт TUN-адаптера (`Get-NetAdapterStatistics`).
- Вычисляет скорость и общий трафик.

#### `MainWindow` (QWidget)
Графический интерфейс:
- Круглая кнопка-переключатель (анимированный `ToggleSwitch`).
- Отображение статуса, ping, скорости, времени сессии.
- Меню (кнопка «⋮») с пунктами «Настройки» и «Статистика».
- Оверлейные диалоги (`OverlayDialog`) для ввода подписки и показа статистики.
- Сохранение/загрузка `settings.json`.

### 3.6. Особенности маршрутизации (Split Tunneling наоборот)

- Весь интернет-трафик направляется через TUN-адаптер (Xray), кроме подключения к IP самого сервера – оно идёт напрямую, чтобы не зацикливать VPN-соединение.
- Используются два маршрута `0.0.0.0/1` и `128.0.0.0/1` (общей суммой покрывающие весь IPv4-диапазон), а не один `0.0.0.0/0`, потому что последний имеет более низкий приоритет, чем маршрут до сервера. Такой приём гарантирует, что трафик к серверу пойдёт через основной интерфейс.

### 3.7. «Починка сети» (Force Fix)

Принудительно завершает `xray.exe`, удаляет маршруты и отключает TUN-адаптер. Используется, если после сбоя VPN не отключается корректно.

## 4. Android-клиент (Kotlin + Jetpack Compose)

### 4.1. Компоненты

| Файл | Назначение |
|------|------------|
| `MainActivity.kt` | Главный экран, логика VPN, UI |
| Зависимость `dev.dev7.lib.v2ray` | Готовая обёртка для V2Ray core (с поддержкой VLESS, REALITY) |
| `vpn_prefs` | SharedPreferences для хранения ссылки, истории, статистики |

### 4.2. Требования

- Android 8.0 (API 26) и выше
- Разрешение `android.permission.INTERNET`
- Разрешение на создание VPN (`VpnService`)

### 4.3. Сборка

1. Открыть проект в **Android Studio**.
2. Добавить в `build.gradle` (Module: app):
   ```gradle
   dependencies {
       implementation 'dev.dev7:libv2ray:6.0.0'
       implementation 'androidx.compose.material3:material3:1.2.0'
       // остальные стандартные зависимости
   }
   ```
3. Собрать APK или AAB.

### 4.4. Архитектура

#### `VPNScreen()` – главный composable

Управляет состоянием:
- `subLink` – текущая ссылка подписки.
- `linkHistory` – последние 5 успешных ссылок.
- `isConnected` – статус VPN.
- `sessionSeconds`, `sessionDown`, `sessionUp` – статистика сессии.
- `totalSeconds`, `totalDown`, `totalUp`, `totalConns` – накопленная статистика (хранится в `SharedPreferences`).

#### Запуск VPN

1. При нажатии на кнопку вызывается `onToggle(true)`.
2. В фоновом корутине вызывается `MarzbanApi.getConfigFromInput(input)`:
   - Если входная строка начинается с `http://` или `https://` → загружается подписка, декодируется base64, извлекается первая VLESS-ссылка.
   - Если начинается с `vless://` → возвращается как есть.
3. Запрашивается разрешение `VpnService.prepare()`. Если нужно – показывается системный диалог через `vpnPermissionLauncher`.
4. После получения разрешения вызывается `startVpnConnection()`, которая строит JSON-конфигурацию через `buildSplitTunnelConfig()` и передаёт её в `V2rayController.startV2ray()`.

#### `buildSplitTunnelConfig(vlessLink)`

Создаёт конфигурацию для V2Ray core:
- Входящий SOCKS-прокси на `127.0.0.1:10808`.
- Исходящий VLESS с параметрами REALITY (если указан `security=reality`).
- Правила маршрутизации (routing):
  - Домены `.ru`, `.su`, `.рф` идут напрямую (`direct`).
  - Приватные IP и IP России (`geoip:ru`) идут напрямую.
  - Весь остальной трафик – через прокси.
- Это реализует **split tunneling**: российские ресурсы не проходят через VPN, остальные – через него.

#### Статистика

- В `LaunchedEffect(isConnected)` каждую секунду:
  - Увеличивается `sessionSeconds`.
  - Через `TrafficStats.getUidRxBytes(Process.myUid())` получаются переданные/полученные байты текущего приложения (так как весь трафик перенаправляется через VPN, эти цифры отражают общий трафик устройства).
- Накопленная статистика сохраняется в `SharedPreferences` при отключении VPN.

#### История ссылок

- При успешном подключении текущая ссылка добавляется в историю (максимум 5).
- В диалоге настроек можно выбрать любую из истории или удалить крестиком.

### 4.5. Важные нюансы

- **Библиотека `dev.dev7.libv2ray`** использует нативные библиотеки V2Ray (v2ray-core) и сама управляет VPNService. Не требует отдельного `xray.exe`.
- Конфигурация передаётся в виде JSON-строки. В ней обязательно должны быть `inbounds`, `outbounds`, `routing`.
- На Android **не требуется** ручная настройка маршрутов – система сама перенаправляет трафик через VPN-интерфейс, который библиотека настраивает автоматически.

## 5. Общая схема передачи данных

```
Пользователь → [Windows: TUN / Android: VPNService] → Xray/V2Ray Core → VLESS+REALITY → Сервер → Интернет
```

1. Приложение генерирует конфигурацию ядра, указывая входящий трафик (TUN или SOCKS) и исходящий (VLESS).
2. Ядро устанавливает соединение с сервером по протоколу REALITY (маскировка под обычный HTTPS).
3. Входящие пакеты инкапсулируются в VLESS, шифруются и отправляются на сервер.
4. Сервер дешифрует, отправляет в интернет, ответ возвращает обратно.
5. Для Windows используется TUN-адаптер на уровне L3, для Android – стандартный VpnService.

## 6. Инструкции по запуску

### 6.1. Для Windows (пользователь)

1. Скачать последний релиз `.exe` (или собрать самостоятельно).
2. Запустить **от имени администратора**.
3. Нажать кнопку «⋮» → «Настройки».
4. Вставить ссылку подписки (или прямую vless://) в поле, нажать «Сохранить».
5. Включить переключатель. Дождаться статуса «Подключено».
6. Для отключения – нажать переключатель снова.

### 6.2. Для Android (пользователь)

1. Установить APK.
2. Открыть приложение, нажать на иконку меню (три полоски) → «Настройки».
3. Вставить ссылку подписки или vless://, нажать «Сохранить».
4. Вернуться на главный экран, нажать на круглую кнопку.
5. При первом запуске система запросит разрешение на VPN – разрешить.
6. Статус изменится на «Подключено». Весь не-российский трафик пойдёт через VPN.

### 6.3. Для разработчиков

#### Windows
- Установить Python 3.8+ и `pip install PySide6`.
- Поместить `xray.exe` и `wintun.dll` в корень проекта (можно скачать с [официального репозитория Xray-core](https://github.com/XTLS/Xray-core/releases)).
- Запуск: `python connector.py`.
- Сборка: `pyinstaller --onefile --windowed --add-data "xray.exe;." --add-data "wintun.dll;." connector.py`.

#### Android
- Клонировать проект, открыть в Android Studio.
- Синхронизировать Gradle.
- Подключить устройство или эмулятор.
- Нажать Run.

## 7. Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| Windows: «Не удалось добавить маршрут до сервера» | Запустить приложение от имени администратора. |
| Windows: TUN-адаптер не создаётся | Убедиться, что `wintun.dll` находится рядом с `xray.exe`. Проверить антивирус. |
| Windows: после закрытия приложения интернет не работает | Использовать «Починка сети» в настройках или вручную выполнить `route delete 0.0.0.0 mask 128.0.0.0` и `route delete 128.0.0.0 mask 128.0.0.0`. |
| Android: не удаётся подключиться (ошибка «VLESS не найден в подписке») | Убедиться, что подписка содержит хотя бы одну ссылку `vless://`. |
| Android: VPN запрашивается каждый раз | Это нормально, система Android требует подтверждения при каждом старте VPN. |
| Ошибка «Xray упал сразу после запуска» | Проверить `xray_console.log`. Чаще всего проблема в неверном формате конфигурации или отсутствии `wintun.dll`. |
