# Техническая документация VPN-клиента для Android (Kotlin + Jetpack Compose)


---

## 1. Введение

Данный документ описывает Android VPN-клиент, использующий протокол **VLESS + REALITY** для обхода блокировок и обеспечения безопасного соединения. В качестве транспорта используется ядро **V2Ray**. Подключение осуществляется по ссылкам-подпискам (содержащим base64-кодированный список ссылок) или по прямой VLESS-ссылке.

Клиент предоставляет графический интерфейс, управление подключением, отображение скорости, трафика и времени сессии.

---

## 2. Общий принцип работы

1. Пользователь вводит **ссылку подписки** (https://…) или **прямую VLESS-ссылку** (vless://…).
2. Приложение извлекает конфигурацию сервера:
   - для подписки: загружает данные, декодирует base64, находит первую VLESS-ссылку.
   - для прямой ссылки: парсит параметры (UUID, адрес, порт, public key, sni, fingerprint и т.д.).
3. Запускается ядро **V2Ray** с динамически сгенерированной конфигурацией.
4. Настраивается **маршрутизация трафика**:
   - Используется системный **VPNService**, который перехватывает весь трафик и направляет его через локальный SOCKS-прокси (порт 10808), запущенный ядром.
5. После отключения VPNService останавливается.

## 3. Android-клиент (Kotlin + Jetpack Compose)

### 3.1. Компоненты

| Файл | Назначение |
|------|------------|
| `MainActivity.kt` | Главный экран, логика VPN, UI |
| Зависимость `dev.dev7.lib.v2ray` | Готовая обёртка для V2Ray core (с поддержкой VLESS, REALITY) |
| `vpn_prefs` | SharedPreferences для хранения ссылки, истории, статистики |

### 3.2. Требования

- Android 8.0 (API 26) и выше
- Разрешение `android.permission.INTERNET`
- Разрешение на создание VPN (`VpnService`)

### 3.3. Сборка

1. Открыть проект в **Android Studio**.
2. Добавить в `build.gradle` (Module: app):
   ```gradle
   dependencies {
       implementation 'dev.dev7:libv2ray:6.0.0'
       implementation 'androidx.compose.material3:material3:1.2.0'
       // остальные стандартные зависимости
   }
   ```
3. Собрать APK или AAB.

### 3.4. Архитектура

#### `VPNScreen()` – главный composable

Управляет состоянием:
- `subLink` – текущая ссылка подписки.
- `linkHistory` – последние 5 успешных ссылок.
- `isConnected` – статус VPN.
- `sessionSeconds`, `sessionDown`, `sessionUp` – статистика сессии.
- `totalSeconds`, `totalDown`, `totalUp`, `totalConns` – накопленная статистика (хранится в `SharedPreferences`).

#### Запуск VPN

1. При нажатии на кнопку вызывается `onToggle(true)`.
2. В фоновом корутине вызывается `MarzbanApi.getConfigFromInput(input)`:
   - Если входная строка начинается с `http://` или `https://` → загружается подписка, декодируется base64, извлекается первая VLESS-ссылка.
   - Если начинается с `vless://` → возвращается как есть.
3. Запрашивается разрешение `VpnService.prepare()`. Если нужно – показывается системный диалог через `vpnPermissionLauncher`.
4. После получения разрешения вызывается `startVpnConnection()`, которая строит JSON-конфигурацию через `buildSplitTunnelConfig()` и передаёт её в `V2rayController.startV2ray()`.

#### `buildSplitTunnelConfig(vlessLink)`

Создаёт конфигурацию для V2Ray core:
- Входящий SOCKS-прокси на `127.0.0.1:10808`.
- Исходящий VLESS с параметрами REALITY (если указан `security=reality`).
- Правила маршрутизации (routing):
  - Домены `.ru`, `.su`, `.рф` идут напрямую (`direct`).
  - Приватные IP и IP России (`geoip:ru`) идут напрямую.
  - Весь остальной трафик – через прокси.
- Это реализует **split tunneling**: российские ресурсы не проходят через VPN, остальные – через него.

#### Статистика

- В `LaunchedEffect(isConnected)` каждую секунду:
  - Увеличивается `sessionSeconds`.
  - Через `TrafficStats.getUidRxBytes(Process.myUid())` получаются переданные/полученные байты текущего приложения (так как весь трафик перенаправляется через VPN, эти цифры отражают общий трафик устройства).
- Накопленная статистика сохраняется в `SharedPreferences` при отключении VPN.

#### История ссылок

- При успешном подключении текущая ссылка добавляется в историю (максимум 5).
- В диалоге настроек можно выбрать любую из истории или удалить крестиком.

### 3.5. Важные нюансы

- **Библиотека `dev.dev7.libv2ray`** использует нативные библиотеки V2Ray (v2ray-core) и сама управляет VPNService. Не требует отдельного исполняемого файла.
- Конфигурация передаётся в виде JSON-строки. В ней обязательно должны быть `inbounds`, `outbounds`, `routing`.
- На Android **не требуется** ручная настройка маршрутов – система сама перенаправляет трафик через VPN-интерфейс, который библиотека настраивает автоматически.

## 4. Общая схема передачи данных

```
Пользователь → [Android: VPNService] → V2Ray Core → VLESS+REALITY → Сервер → Интернет
```

1. Приложение генерирует конфигурацию ядра, указывая входящий трафик (SOCKS) и исходящий (VLESS).
2. Ядро устанавливает соединение с сервером по протоколу REALITY (маскировка под обычный HTTPS).
3. Входящие пакеты инкапсулируются в VLESS, шифруются и отправляются на сервер.
4. Сервер дешифрует, отправляет в интернет, ответ возвращает обратно.
5. На Android используется стандартный VpnService, который перенаправляет весь трафик через локальный SOCKS-прокси.

## 5. Инструкции по запуску

### 5.1. Для пользователя Android

1. Установить APK.
2. Открыть приложение, нажать на иконку меню (три полоски) → «Настройки».
3. Вставить ссылку подписки или vless://, нажать «Сохранить».
4. Вернуться на главный экран, нажать на круглую кнопку.
5. При первом запуске система запросит разрешение на VPN – разрешить.
6. Статус изменится на «Подключено». Весь не-российский трафик пойдёт через VPN.

### 5.2. Для разработчиков Android

- Клонировать проект, открыть в Android Studio.
- Синхронизировать Gradle.
- Подключить устройство или эмулятор.
- Нажать Run.

## 6. Возможные проблемы и решения (Android)

| Проблема | Решение |
|----------|---------|
| Не удаётся подключиться (ошибка «VLESS не найден в подписке») | Убедиться, что подписка содержит хотя бы одну ссылку `vless://`. |
| VPN запрашивается каждый раз | Это нормально, система Android требует подтверждения при каждом старте VPN. |
| Подключение есть, но некоторые сайты не открываются | Проверьте правила split tunneling: возможно, нужный домен попал в исключения. Измените конфигурацию routing. |
| Приложение падает при старте VPN | Проверьте корректность VLESS-ссылки (особенно параметры `security=reality`, `pbk`, `sid`). Посмотрите логи через `adb logcat`. |
| Нет интернета после отключения VPN | Перезапустите приложение или перезагрузите устройство – это сбрасывает состояние VPNService. |
